principles

how the meadow stays usable over time

what gets built

  • narrow tools for recurring annoyances
  • simple inputs, obvious outputs
  • stable dependencies
  • pages that can rank by intent, not promotion

what gets avoided

  • features that create support burden
  • platform lock-in where possible
  • anything that needs a content treadmill
  • products that only work with constant attention

a small public queue so the direction stays coherent
